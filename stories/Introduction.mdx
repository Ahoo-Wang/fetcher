        <li><strong>🧭 Path & Query Parameters</strong>: Built-in support for path (`{id}`/`:id`) and query parameters</li>
        <li><strong>🔗 Interceptor System</strong>: Request, response, and error interceptors with ordered execution for flexible middleware patterns</li>
        <li><strong>⏱️ Timeout Control</strong>: Configurable request timeouts with proper error handling</li>
        <li><strong>🔄 Fetch API Compatible</strong>: Fully compatible with the native Fetch API</li>
        <li><strong>🛡️ TypeScript Support</strong>: Complete TypeScript definitions for type-safe development</li>
        <li><strong>🧩 Modular Architecture</strong>: Lightweight core with optional extension packages</li>
        <li><strong>📦 Named Fetcher Support</strong>: Automatic registration and retrieval of fetcher instances</li>
        <li><strong>⚙️ Default Fetcher</strong>: Pre-configured default fetcher instance for quick start</li>
      </ul>

      <Divider />

      <Typography.Title level={3}>🎨 @ahoo-wang/fetcher-decorator - Declarative APIs</Typography.Title>
      <Typography.Paragraph>

      </Typography.Paragraph>
      <ul style={{ paddingLeft: '20px' }}>
        <li><strong>🎨 Clean API Definitions</strong>: Define HTTP services using intuitive decorators</li>
        <li><strong>🧭 Automatic Parameter Binding</strong>: Path, query, header, and body parameters automatically bound</li>
        <li><strong>⏱️ Configurable Timeouts</strong>: Per-method and per-class timeout settings</li>
        <li><strong>🔗 Fetcher Integration</strong>: Seamless integration with Fetcher's named fetcher system</li>
        <li><strong>⚡ Automatic Implementation</strong>: Methods automatically implemented with HTTP calls</li>
        <li><strong>📦 Metadata System</strong>: Rich metadata support for advanced customization</li>
      </ul>

      <Divider />

      <Typography.Title level={3}>🎯 @ahoo-wang/fetcher-eventbus - Event Bus System</Typography.Title>
      <Typography.Paragraph>
        A TypeScript event bus library providing multiple implementations for handling events: serial execution,
        parallel
        execution, and cross-tab broadcasting.
      </Typography.Paragraph>
      <ul style={{ paddingLeft: '20px' }}>
        <li><strong>🔄 Serial Execution</strong>: Execute event handlers in order of priority</li>
        <li><strong>⚡ Parallel Execution</strong>: Run event handlers concurrently for better performance</li>
        <li><strong>🌐 Cross-Tab Broadcasting</strong>: Broadcast events across browser tabs using BroadcastChannel API</li>
        <li><strong>📦 Generic Event Bus</strong>: Manage multiple event types with lazy loading</li>
        <li><strong>🔧 Type-Safe</strong>: Full TypeScript support with strict typing</li>
        <li><strong>🧵 Async Support</strong>: Handle both synchronous and asynchronous event handlers</li>
        <li><strong>🔄 Once Handlers</strong>: Support for one-time event handlers</li>
        <li><strong>🛡️ Error Handling</strong>: Robust error handling with logging</li>
